	\SubsectionBox{Background from C1--C7}
	In C1--C4, you modeled continuous random variables with linear and piecewise probability density functions (PDFs), ensuring non-negativity and total area equal to \(1\). In C5--C7, you worked with the normal distribution, standard-normal calculations, and inverse-normal reasoning. In this activity, you will extend that work into a computational project that combines probability modeling, visualization, data analysis, and evidence-based reporting.
	
	\SubsectionBox{Task overview}
	Create one webpage (HTML + CSS + JavaScript) that performs continuous probability modeling in two connected modes, and use it to support a short written report:
	\begin{enumerate}
		\item \textbf{Simulation mode (C8):} users modify distribution parameters, graph PDFs, and calculate probabilities under the curve.
		\item \textbf{Data modeling mode (C9):} users upload or enter data, estimate distribution parameters, and compare fitted PDFs with empirical histograms.
		\item \textbf{Report work (C10):} students use real-life data and real-world sources to write a short evidence-based report.
	\end{enumerate}
	The page must be functional, not only visual. Do not submit solved exercises, completed answer sheets, or screenshots as a replacement for functionality.

	\SubsectionBox{Mandatory requirements for C8}
	Your C8 implementation must include interactive PDF graphs for the following distributions:
	\begin{enumerate}
		\item Uniform distribution.
		\item Triangular distribution.
		\item Increasing or decreasing linear distribution.
		\item Piecewise distribution.
		\item Normal distribution.
	\end{enumerate}
	
	For each distribution, your webpage must:
	\begin{enumerate}
		\item Allow the user to enter or modify all defining parameters.
		\item Generate the corresponding PDF from the parameters.
		\item Display the PDF graph interactively.
		\item Compute probabilities under the curve for:
		\begin{itemize}
			\item probability below a point,
			\item probability above a point,
			\item probability between two points.
		\end{itemize}
		\item Visually shade the selected probability region.
		\item Display the numerical probability value.
		\item Update the graph and values whenever parameters or bounds change.
		\item Include a hyperlink to the webpage deployed through GitHub Pages or another accessible public/shared link.
		\item Include a visible \textbf{AI development record} section in the deployed webpage with hyperlinks to the AI conversation(s) used.
	\end{enumerate}
	
	\SubsectionBox{Distribution-specific C8 conditions}
	Your webpage must satisfy the following conditions:
	\begin{enumerate}
		\item \textbf{Uniform:} input lower endpoint and upper endpoint; generate a constant PDF.
		\item \textbf{Triangular:} input lower endpoint, upper endpoint, and mode; generate a triangular PDF.
		\item \textbf{Linear:} define an increasing or decreasing linear PDF on a finite interval; ensure total area under the PDF is \(1\).
		\item \textbf{Piecewise:} define at least one piecewise PDF using intervals and formulas; graph all pieces correctly.
		\item \textbf{Normal:} input mean and standard deviation; graph the normal PDF.
	\end{enumerate}
	
	\SubsectionBox{Mandatory requirements for C9}
	Your C9 implementation must use AI-assisted programming to create a tool that receives data and estimates distribution parameters.
	
	The tool must support at least the following fitted models:
	\begin{enumerate}
		\item Uniform distribution.
		\item Triangular distribution.
		\item Linear distribution.
		\item Piecewise uniform distribution.
		\item Normal distribution.
	\end{enumerate}
	
	Your webpage must allow the user to upload or input data and choose which distribution to fit. For each fit, display:
	\begin{enumerate}
		\item estimated parameters,
		\item fitted PDF,
		\item histogram of uploaded data,
		\item histogram and fitted PDF on the same graph,
		\item short interpretation of whether the fitted distribution appears reasonable,
		\item a hyperlink to the webpage deployed through GitHub Pages or another accessible public/shared link.
		\item visible hyperlinks in the deployed webpage to the AI conversation(s) used.
	\end{enumerate}

	\SubsectionBox{C10 modeling requirements}
	For the C10 report, analyze real-life data and complete the following:
	\begin{enumerate}
		\item Use the three provided CSV datasets to model continuous data, and you may add real-life datasets as additional evidence.
		\item For each provided dataset, display the dataset histogram.
		\item For each dataset, fit at least three continuous probability models.
		\item At minimum, include:
		\begin{itemize}
			\item piecewise uniform model,
			\item linear model,
			\item normal model.
		\end{itemize}
		\item Plot each fitted PDF on the same graph as the histogram.
		\item Compare visually which model appears to fit better.
		\item Explain one strength and one limitation of each model for the dataset.
	\end{enumerate}
	
	\SubsectionBox{C10 probability-calculation requirements}
	For each dataset and each fitted model, calculate:
	\[
	P(\mu-\sigma < X < \mu+\sigma)
	\]
	and
	\[
	P(X<\mu-2\sigma\;\text{or}\;X>\mu+2\sigma).
	\]
	
	Use probabilities from the fitted model, meaning area under the model PDF, not only point-counting from the raw data.
	
	\SubsectionBox{APA format requirements for C10}
	The report must use APA style in the following way:
	\begin{enumerate}
		\item Use clear academic headings, such as title, introduction, methodology, results, discussion, conclusion, and references.
		\item Use author--year in-text citations when making contextual claims. Example: \texttt{(Author, year)}.
		\item Include a references section titled \textbf{References}.
		\item List the references in alphabetical order by the first author's last name or by the institutional author.
		\item Each reference must include the author or institution, year, title, source name, and URL or DOI when available.
		\item Use APA 7th edition style for web sources. Example format: Author or Institution. (Year). \textit{Title of page or report}. Website or Publisher. URL
		\item Do not include sources in the references section if they are not cited inside the report.
	\end{enumerate}
