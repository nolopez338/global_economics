\SubsectionBox{Background from C1--C7}
In C1--C7 you already practiced the full statistical reasoning cycle: writing \(H_0\) and \(H_1\), selecting the tail direction, calculating test statistics, making decisions with \(\alpha\), interpreting errors, and determining minimum sample sizes. In this activity, you will transform that reasoning into a working webpage.

\SubsectionBox{Task overview}
Create one webpage (HTML + CSS + JavaScript) that performs hypothesis testing in two connected modes:
\begin{enumerate}
\item \textbf{Manual mode (C8):} users set parameters and visualize the p-value on an interactive normal curve.
\item \textbf{CSV mode (C9):} users upload two-group data and the webpage computes the hypothesis test automatically.
\end{enumerate}
The page must be functional, not only visual. Do not submit worked solutions, completed answer sheets, or screenshots as a replacement for functionality.

\SubsectionBox{Mandatory requirements for C8}
Your C8 implementation must include all of the following:
\begin{itemize}
\item Input controls using both \textbf{sliders} and \textbf{input boxes}.
\item Test-direction selector: left-tailed, right-tailed, two-tailed.
\item Significance level \(\alpha\).
\item Test statistic \(z\), or enough parameters to calculate \(z\).
\item Null value.
\item Sample statistic.
\item Standard deviation or standard error.
\item Sample size.
\end{itemize}

The output panel/graph must show:
\begin{itemize}
\item The normal curve.
\item The observed test statistic.
\item The shaded p-value region.
\item The numerical p-value.
\item The critical value or critical region.
\item The statistical decision: reject \(H_0\) or fail to reject \(H_0\).
\item A short contextual conclusion consistent with the decision.
\end{itemize}

\SubsectionBox{Mandatory requirements for C9}
Your C9 implementation must include all of the following:
\begin{itemize}
\item CSV upload control.
\item Reading data from two groups.
\item Detection of group columns, or user selection of group columns.
\item Computation of summary statistics for each group.
\item Selection of benchmark group and test group.
\item Test-direction selector: left-tailed, right-tailed, two-tailed.
\item Automatic execution of the hypothesis test.
\item Display of test statistic, p-value, decision, and contextual conclusion.
\item Display of the p-value on the same type of normal-curve graph used in C8.
\end{itemize}

\SubsectionBox{Required CSV example files}
Use and reference these attached files:
\begin{itemize}
\item \texttt{G11\_T3\_L4\_C8C9\_example\_mean\_data.csv}
\item \texttt{G11\_T3\_L4\_C8C9\_example\_proportion\_data.csv}
\end{itemize}

Expected structure:
\begin{itemize}
\item \textbf{Mean-based file:} two group columns containing numerical observations (for example counts, times, or scores).
\item \textbf{Proportion-based file:} two group identifiers plus a binary success variable (0/1) for each observation.
\end{itemize}

These two files must run correctly in your webpage and be used to validate your implementation.

\SubsectionBox{Evaluation checklist (validation requirements)}
Your work is valid only if all checks pass:
\begin{enumerate}
\item The graph updates when parameters change.
\item The p-value updates when parameters change.
\item The decision updates when parameters change.
\item The CSV files are read correctly.
\item The test is computed correctly from uploaded data.
\item Both attached CSV examples work in the webpage.
\item The contextual conclusion matches the displayed decision.
\end{enumerate}
