	\SubsectionBox{Background from C1--C7}
	In C1--C7 you already practiced the full statistical reasoning cycle: writing \(H_0\) and \(H_1\), selecting the tail direction, calculating test statistics, making decisions with \(\alpha\), interpreting errors, and determining minimum sample sizes. In this activity, you will transform that reasoning into a working webpage and use the results to write a short report.
	
	\SubsectionBox{Task overview}
	Create one webpage (HTML + CSS + JavaScript) that performs hypothesis testing in two connected modes, and use it to support a short written report:
	\begin{enumerate}
		\item \textbf{Manual mode (C8):} users set parameters and visualize the p-value on an interactive normal curve.
		\item \textbf{CSV mode (C9):} users upload two-group data and the webpage computes the hypothesis test automatically.
		\item \textbf{Report work (C10):} students write a short report using the webpage results and three real-world sources.
	\end{enumerate}
	The page must be functional, not only visual. Do not submit worked solutions, completed answer sheets, or screenshots as a replacement for functionality.

	\SubsectionBox{Mandatory requirements for C8}
	Your C8 implementation must include all of the following:
	\begin{enumerate}
		\item Input controls using both \textbf{sliders} and \textbf{input boxes}.
		\item Test-direction selector: left-tailed, right-tailed, two-tailed.
		\item Significance level \(\alpha\).
		\item Two input modes for calculating the test statistic \(z\):
		\begin{itemize}
			\item \textbf{Proportion setting mode:} the webpage must calculate \(z\) using the benchmark proportion, sample proportion, and sample size.
			\item \textbf{Mean setting mode:} the webpage must calculate \(z\) using the benchmark mean, sample mean, standard deviation or standard error, and sample size.
		\end{itemize}
		Each parameter in both modes must be editable using both sliders and input boxes.
		\item Null value, hypothesis or benchmark.
		\item Sample statistic.
		\item A hyperlink to the webpage deployed through GitHub Pages.
	\end{enumerate}
	
	The output panel/graph must show:
	\begin{enumerate}
		\item The normal curve.
		\item The observed test statistic.
		\item The shaded p-value region.
		\item The numerical p-value.
		\item The critical value or critical region.
		\item The statistical decision: reject \(H_0\) or fail to reject \(H_0\).
		\item A short contextual conclusion consistent with the decision.
	\end{enumerate}
	
	\SubsectionBox{Mandatory requirements for C9}
	Your C9 implementation must include all of the following:
	\begin{enumerate}
		\item CSV upload control.
		\item Reading data from two groups.
		\item Detection of group columns, or user selection of group columns.
		\item Computation of summary statistics for each group.
		\item Selection of benchmark group and test group.
		\item Test-direction selector: left-tailed, right-tailed, two-tailed.
		\item Automatic execution of the hypothesis test.
		\item Display of test statistic, p-value, decision, and contextual conclusion.
		\item Display of the p-value on the same type of normal-curve graph used in C8.
		\item A hyperlink to the webpage deployed through GitHub Pages.
	\end{enumerate}

	\SubsectionBox{Mandatory requirements for C10}
	Your C10 work must be a short report based on hypothesis testing and real-world context. The report must follow APA style and include all of the following:
	\begin{enumerate}
		\item A title.
		\item A real-world context connected to the hypothesis test.
		\item At least \textbf{three real-world sources}.
		\item A clear research question.
		\item The null and alternative hypotheses.
		\item The test direction and significance level \(\alpha\).
		\item The test statistic, p-value, and statistical decision.
		\item A contextual conclusion consistent with the decision.
		\item A short discussion of what the result means, and the errors that can be incurred in the real-world context.
		\item APA-style in-text citations when using information from the sources.
		\item A references section in APA format.
	\end{enumerate}
	
	\SubsectionBox{APA format requirements for C10}
	The report must use APA style in the following way:
	\begin{enumerate}
		\item Use clear academic headings, such as title, introduction, methodology, results, discussion, conclusion, and references.
		\item Use author--year in-text citations when making contextual claims. Example: \texttt{(Author, year)}.
		\item Include a references section titled \textbf{References}.
		\item List the references in alphabetical order by the first author's last name or by the institutional author.
		\item Each reference must include the author or institution, year, title, source name, and URL or DOI when available.
		\item Use APA 7th edition style for web sources. Example format: Author or Institution. (Year). \textit{Title of page or report}. Website or Publisher. URL
		\item Do not include sources in the references section if they are not cited inside the report.
	\end{enumerate}
	
	\SubsectionBox{Real-world source requirements for C10}
	The three real-world sources must satisfy the following:
	\begin{enumerate}
		\item They must be connected to the context of the report.
		\item They must be reliable sources, such as official statistics sites, academic publications, institutional reports, or reputable data portals.
		\item They must be cited inside the report when making contextual claims.
		\item They must appear in the references section.
	\end{enumerate}
